## Component Descriptions and Responsibilities

### REMS Web Interface (Presentation Tier)
The REMS Web Interface serves as the primary interaction point for users. It provides features such as dashboards, application forms, guidance panels, and status tracking. It captures user inputs, including application data and document uploads, and forwards all requests to the backend. It is responsible only for presentation and interaction logic, and does not handle business processing or data storage.

### Authentication & Authorisation
This component is responsible for verifying user identity and enforcing access control. It integrates with the external Identity Provider (University SSO) to authenticate users and ensures that only authorised users can access or modify system resources. It also manages roles and permissions and filters all incoming requests before they reach the core system services.

### Application Management Service
The Application Management Service is the central coordinator of the system. It manages the full lifecycle of an ethics application, including creation, draft saving, auto-save, submission, revision, and status updates. It orchestrates interactions between other services, ensuring that validation, document handling, notifications, and audit logging are triggered at the appropriate stages. It contains the core business logic of the system.

### Validation Service
The Validation Service is responsible for ensuring that applications meet all submission requirements. It performs completeness checks and validates required fields and supporting documents before submission or resubmission. It returns detailed error information to support user feedback and correction.

### Document Management & Versioning Service
This component manages all aspects of document handling. It supports document upload, storage, retrieval, and version control. It maintains metadata such as version history and ensures that previous versions of documents can be accessed when needed, providing consistency and traceability.

### Notification Service
The Notification Service handles communication with users. It generates notifications for key events such as submission confirmation, revision requests, and status updates. Notifications are delivered both within the system and via email through an external email service.

### Audit Trail Service
The Audit Trail Service records all significant system actions, including application creation, modifications, submissions, and status changes. It maintains append-only, timestamped records to ensure traceability, accountability, and compliance with audit requirements.

### Relational Database (Data Tier)
The relational database stores all structured data within the system, including applications, user information, permissions, audit logs, and notification records. It ensures data integrity, consistency, and efficient retrieval of system data.

### File / Object Storage
This component is responsible for storing uploaded documents and file content. It supports versioned storage and works in conjunction with the Document Management Service to manage document access and retrieval.

### External Systems

#### Identity Provider (University SSO)
Provides authentication services and verifies user identity during login.

#### External Email Service (SMTP)
Handles the delivery of email notifications generated by the system.
