# 4401_project

## Stakeholder Identification and Initial Acquisition

Identifying Stakeholders:

Interactive Perspective: Principal Researcher, Research Assistant, Ethics Committee Member, Administrative Secretary.

Indirect Perspective: University Legal Counsel (Privacy Compliance), IT Maintenance Department.

Writing User Stories:

Format: As a <role>, I want <goal> so that <benefit>.

Tip: Focus on "business value" rather than specific interfaces.

Example: "As a researcher, I want to see my application status in the system so I can plan when to start my experiment."

## Interview Preparation

Selecting Target Roles: It is recommended to select "Committee Members" or "Administrative Staff," as their business processes (approval workflow, conflict management) are the most challenging.

Designing Questions: Each question must have a Rationale.

Correct Example: "If a committee member discovers that the applicant is their relative, how does the current offline process handle 'conflict of interest'?"

Rationale: Used to define whether the system needs "avoidance logic" and how to design the approval allocation algorithm.

## the following questions for the Principle Researcher （Md Faraz Kabir Khan）
1. Preferred application form completion method
 
Question:
What is your preferred method for completing application forms: directly within the portal, by downloading and completing a document offline, or by having access to both options?
 
Justification:
The current process requires researchers to complete application forms as separate documents and submit them by email. Although a portal-based form may improve efficiency and accessibility, some users may still prefer an offline document-based approach. This question seeks to determine user preference so that the system design aligns with stakeholder needs.
 
2. Access for co-researchers to upload supporting documentation
 
Question:
Should co-researchers associated with a project be permitted to upload supporting documents to the application?
 
Justification:
Research applications are often prepared collaboratively. Enabling co-researchers to upload relevant supporting documentation may improve efficiency and reduce the workload of the principal researcher. As this requirement is not explicitly addressed in the project description, clarification from stakeholders is required.
 
3. Preferred application progress notification method
 
Question:
How would you prefer to receive updates on the progress of your application: through email notifications, a status dashboard, or both?
 
Justification:
A lack of transparency and delays in the application process have been identified as key concerns among researchers. Regular progress updates may help address these issues. Since the project description does not define how such updates should be delivered, this question aims to identify the most appropriate communication method.
 
4. Preferred communication mechanism for application-related interactions
 
Question:
What type of communication channel would you prefer for application-related interactions, such as clarification requests, updates, and decisions: an internal thread-based system or a formal external group channel such as Microsoft Teams?
 
Justification:
The project description highlights the absence of a formal communication mechanism for application-related correspondence. However, no preferred communication model has been specified. This question is intended to identify the communication approach that best meets stakeholder expectations.
